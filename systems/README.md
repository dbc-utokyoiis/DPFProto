# Benchmark Systems

Cross-system benchmark suite for TPC-H and SSB (Star Schema Benchmark) over Parquet files.

## Directory Structure

```
systems/
  common.sh               # Shared configuration (paths, scale factors, layout)
  check_gds.sh            # Verify GPUDirect Storage setup (called by GPU benchmarks)
  run_all.sh              # Setup + Load + Benchmark in one shot
  01_setup.sh             # Setup all systems
  02_load.sh              # Load Parquet data via DuckDB
  03_run_benchmark.sh     # Run benchmarks across all systems
  duckdb/
    setup.sh              # Install DuckDB CLI (pinned version)
    load.sh               # Generate Parquet from raw TBL files (BENCHMARK=tpch|ssb|all)
    run_benchmark.sh      # Run DuckDB queries over Parquet
  polars/
    setup.sh              # Create venv with polars[gpu], kvikio, pyarrow
    run_benchmark.sh      # Set GDS env vars, invoke bench.py
    bench.py              # TPC-H + SSB queries via Polars LazyFrame API (GPU engine)
  spark-rapids/
    setup.sh              # Download Spark 4.0.1 + RAPIDS 25.12.0 JAR
    conf/                 # Spark config templates (spark-defaults.conf, log4j2.properties, ...)
    run_benchmark.sh      # Entry point (dispatches to run_ssb/run_tpch based on BENCHMARK)
    run_ssb_benchmark.sh  # SSB benchmark via spark-submit (GPU/CPU)
    run_tpch_benchmark.sh # TPC-H benchmark via spark-submit (GPU/CPU)
    ssb_benchmark.py      # SSB query driver (PySpark SQL)
    tpch_benchmark.py     # TPC-H query driver (PySpark SQL)
    ssb_queries/          # SSB SQL files (q11..q43)
    queries/              # TPC-H SQL files (q1,q3,q5,q6,q13,q16)
```

## Prerequisites

```bash
sudo apt-get update && sudo apt-get install -y \
  curl unzip wget \
  python3 python3-venv python3-pip \
  openjdk-21-jdk
```

NVIDIA driver, CUDA Toolkit, and GPUDirect Storage must be installed separately via NVIDIA's repositories.

## Configuration

All systems share `common.sh` for consistent settings:

| Variable   | Description          | Default    |
|------------|----------------------|------------|
| `LAYOUT`   | Data layout          | `sideways` |
| `SF`       | Scale factor (TPC-H / SSB) | `100` |
| `NTHREADS` | Thread count         | `$(nproc)` |
| `MEMORY`   | Memory limit (DuckDB/Polars) | `384GB` |
| `SPARK_DRIVER_MEMORY` | Spark JVM heap | `384g` |
| `SPARK_EXPLAIN` | RAPIDS SQL explain (`NONE` or `ALL`) | `NONE` |

## Workflow

### Run everything at once

```bash
cd systems/

# All benchmarks, SF=100 (default)
bash run_all.sh

# SSB only, SF=1
SF=1 BENCHMARK=ssb bash run_all.sh
```

### Run each step individually

```bash
cd systems/
bash 01_setup.sh           # Setup all systems (DuckDB, Polars, Spark-Rapids)
bash 02_load.sh            # Generate Parquet via DuckDB
bash 03_run_benchmark.sh   # Run benchmarks on all systems
```

### Run a single system

```bash
cd systems/

# DuckDB
bash duckdb/setup.sh
BENCHMARK=ssb SF=100 bash duckdb/load.sh
BENCHMARK=ssb SF=100 bash duckdb/run_benchmark.sh

# Polars (GPU)
bash polars/setup.sh
BENCHMARK=ssb SF=100 bash polars/run_benchmark.sh

# Spark-RAPIDS (GPU)
bash spark-rapids/setup.sh
BENCHMARK=ssb SF=100 bash spark-rapids/run_ssb_benchmark.sh --mode gpu

# Spark-RAPIDS with explain diagnostics
SPARK_EXPLAIN=ALL BENCHMARK=ssb SF=1 bash spark-rapids/run_ssb_benchmark.sh --mode gpu 2>&1 | tee logs/spark-rapids/ssb/sf1/run.log
```

## Dry Run

All scripts support `--dry-run` to show what would be executed without actually running:

```bash
bash duckdb/setup.sh --dry-run
BENCHMARK=ssb SF=100 bash duckdb/load.sh --dry-run
bash duckdb/run_benchmark.sh --dry-run
```

## Data Flow

DuckDB converts raw TBL files into Parquet. All other systems (Polars, Spark-Rapids) read the same Parquet files directly — no separate load step is needed.

## Reference Answers

Each benchmark run verifies query results against reference answers in `answers/`. These are generated by DuckDB from the raw TBL files:

```bash
# TPC-H (from repo root)
bash scripts/tpch/gen_answers.sh -s 1       # SF=1
bash scripts/tpch/gen_answers.sh -s 100     # SF=100

# SSB (from repo root)
bash scripts/ssb/gen_answers.sh -s 1        # SF=1
bash scripts/ssb/gen_answers.sh -s 100      # SF=100
```

Output structure:

```
answers/
  tpch/sf100/
    q1.csv, q3.csv, q5.csv, q6.csv, q13.csv, q16.csv
  ssb/sf100/
    q11.csv, q12.csv, ..., q43.csv
```

The `--no-load` flag skips the CSV-to-Parquet conversion step (useful when intermediate Parquet already exists):

```bash
bash scripts/ssb/gen_answers.sh -s 100 --no-load
```

**Note:** The gen_answers scripts use system-installed `duckdb`, not the cached binary in `systems/duckdb/bin/`.
